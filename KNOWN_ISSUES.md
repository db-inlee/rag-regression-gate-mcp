# KNOWN ISSUES — 표 추출 (Phase 1.1)

## ISSUE-1: 광폭(컬럼 多) 명세표에서 숫자·단어가 공백으로 분할됨

### 증상
컬럼 수가 매우 많은 명세표(종속기업 명세, 타법인출자 현황 상세, 공정가치 공시 등)에서
셀 내부의 숫자 끝자리 또는 단어가 공백으로 쪼개진다.

| 원래 값 | 추출 결과 (깨짐) |
|---|---|
| `11,657,872` | `11,657,87 2` |
| `16,009,547` | `16,009,54 7` |
| `65,328,568` | `65,328,5 68` |
| `경영참여` | `경영참 여` |
| `지분율` (헤더) | `지분 율` |

### 원인 (추정)
표의 컬럼이 매우 많아 각 컬럼 폭이 좁을 때, pdfplumber의 열 경계 추정에서
셀 폭보다 토큰이 넓어 마지막 글자/숫자가 인접 가상 컬럼 쪽으로 분리된다.
**표의 행/열 구조 자체는 보존되며, 셀 내부 텍스트만 공백으로 분할**된다.

### 영향 범위 (2026-06-16 측정, 9개 파일)
분할셀 추정은 `grep -oE ",[0-9]+ [0-9]{1,2} \|"` 기반 근사치(과소 집계 가능).

| 파일 | 영향받은 표 수 | 분할셀(근사) |
|---|---|---|
| 삼성전자_2023 | 25 | 594 |
| 삼성전자_2024 | 16 | 243 |
| 삼성전자_2025 | 14 | 244 |
| 하이닉스_2023 | 36 | 328 |
| 하이닉스_2024 | 28 | 162 |
| 하이닉스_2025 | 26 | 260 |
| 현대자동차_2023 | 2 | 10 |
| 현대자동차_2024 | 8 | 52 |
| 현대자동차_2025 | 9 | 51 |

> ⚠️ 당초 추정("파일당 0~3건")과 달리, 실제로는 **파일당 표 2~36개 / 분할셀 약 10~594개**로
> 범위가 훨씬 넓다. 다만 거의 전부가 소수의 **광폭 명세/공시표**에 집중된다.

### 대표 예시
- **삼성전자_2023 `p.507` / `Table p507-1` — 타법인출자 현황(상세)** (16+열)
  - `11,657,87 2`(→11,657,872), `16,009,54 7`(→16,009,547), `경영참 여`(→경영참여)
- **삼성전자_2023 `p.87` / `Table p87-2` — 전체 종속기업 자산/부채 명세** (232개사 전치, 28+열)
  - `65,328,5 68`(→65,328,568), `7,266,21 3`(→7,266,213)
- **하이닉스_2023 `p.52` / `Table p52-1` — 연결 자본변동표** (자본 구성요소 多)
  - `51,909,0 97`(→51,909,097), `62,191,0 58`(→62,191,058)

### 핵심 재무제표 영향 없음 (검증됨)
사용자가 지정한 **3대 핵심 재무제표(손익계산서 · 재무상태표 · 현금흐름표)** 의 표 블록에서
분할셀 **0건** (9개 파일 전부). 표 직전 본문이 해당 제목을 가진 표만 대상으로 strict 검사.

> 단, **자본변동표**(역시 기본 재무제표지만 컬럼이 넓음)에는 소수(하이닉스 2개 파일에서 1~2건) 발생.
> 사용자가 명시한 3종에는 포함되지 않으나, 폭넓은 표라는 동일 원인이므로 함께 기록.

### 상태 / 향후 방향 (참고, 미착수)
- 현재: **기록만**. 핵심 재무제표 QA에는 영향이 없어 Phase 1.1 범위에서는 차단 요인 아님.
- 향후 후보(미결정): pdfplumber `table_settings`의 컬럼 경계 전략 조정, 셀 내부 공백 정규화
  후처리(숫자 그룹 재결합), 또는 광폭표 전용 추출 경로. **승인 전 구현 금지.**
