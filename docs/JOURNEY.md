# JOURNEY — 검증으로 실제 잡은 결함들

> 이 프로젝트의 핵심 가치는 "동작하는 코드"가 아니라 **"검증으로 결함을 잡고 정직하게 고친 과정"**이다.
> 각 단계마다 가설을 세우고 → 실제 데이터/로그로 교차검증 → 결함을 찾아 수정했다. 아래는 happy-path가
> 아니라 **검증이 없었으면 그대로 새어나갔을** 결함과 그 수정의 기록이다. (데모를 위한 결과 조작은 없음.)

## Phase 2 — 채점 / judge / 귀인

| 검증으로 발견한 결함 | 근거 | 수정 |
|---|---|---|
| **채점 false negative** — 모델 "14조 2,396억원"(=14,239,600) vs gold 14,239,592를 오답 처리 | **case_034** baseline 교차검증 | table_value 정확일치 → **상대 허용오차 ±0.1%** (case_031 0.14%·case_002 0.8%는 여전히 incorrect) |
| **judge 무효 probe로 정확도 deflation** — '맞는 내용 덧붙인' 변형이 여전히 정답이라 거짓 FP | **case_082·089**, judge_accuracy 0.933 | neg_subtle을 '진짜 사실 어긋남'으로만 + 생성후 자체 모순점검 → **0.933 → 0.987**(잔존 FP는 case_093 용어교체 1건) |
| **암기/운 정답이 RAG 점수 부풀림** — 맞았지만 정답값이 검색근거에 없음 | **case_034** | `grounded_correct` vs `unsupported_correct` 분리, answerable_accuracy 헤드라인은 grounded 기준 |
| **가짜 retrieval_miss** — gold 표는 strict-miss지만 동등표(요약재무)에 정답값 존재 | **case_038** | `retrieval_success`를 strict(표) + value_present(값 어디든) **이중 정의**로 분리 보고 |
| **파생값(증가율) groundedness 과소** — %는 셀에 없어 무조건 unsupported | case_043/046 | 원천 셀 2개가 검색됐으면 grounded 인정(case_043 grounded, case_046은 미검색 → unsupported 유지) |

## Phase 3 — 회귀 판정 / 게이트

| 검증으로 발견한 결함 | 근거 | 수정 |
|---|---|---|
| **노이즈밴드 0이면 1케이스 플립을 회귀로 오판 위험** | 노이즈밴드 측정 std=0 | 측정값은 정직한 0 유지, **판정에서만 floor `max(밴드, ±1 case)`** |
| **거짓경보 검증 필요** | 같은 config 두 실행(r1 vs r2) | delta 전부 0, CI [0,0] → **PASS(거짓경보 0건)** 확인 |
| **검출기가 무음 no-op일 위험** | 합성 회귀 주입 검증 | 12케이스 강등 시 retrieval_miss CI [+6,+18]로 FAIL → 변별력 확인 |
| **게이트 WARN 문구에 "유의" 오표기** | WARN인데 "유의 하락"으로 출력 | WARN은 "유의" 제거 + "[경계]" 표기 |

| 검증으로 발견한 결함/사실 | 근거 | 처리 |
|---|---|---|
| **음수 기준 증가율** — 적자 기준이라 6066% 등 무의미 | **case_052**(하이닉스 매출총이익), **case_044**(영업이익, 같은 결함) | growth는 **기준연도 값>0**만; 둘 다 양수기준으로 교체 |

| **reranker가 "헛개선"** — 직관과 달리 유의 개선 없음 | B 게이트 WARN | **가설과 다른 결과 그대로 보고** — 게이트가 헛개선 인정 안 함(가드의 존재 이유) |
| **중립(overlap)도 1케이스 경계 흔들림** | C 게이트 WARN | FAIL 오판은 floor가 막음; "완전 PASS 아님"을 정직 기록(floor ±2 트레이드오프 명시) |

## 메타 교훈

- **검증 단위가 잘게 쪼개져 있을수록 결함이 드러난다**: 케이스 단위(case_034/038/052…)로 교차검증했기에 집계 점수에 묻힐 결함을 잡았다.
- **가설과 다른 결과는 결함의 신호이거나 정직성의 기회다**: chunk400 WARN·reranker 헛개선은 꾸미지 않고 그대로 보고했고, 그게 오히려 게이트의 분별력을 증명했다.
- **"검증 도구 자체도 검증한다"**: judge를 gold로 검증(0.987)하고, 게이트를 거짓경보 테스트(0건)로 검증했다 — 측정하는 것이 못 믿을 만하면 측정 전체가 무의미하므로.
